# Global Rules

## Communication Language

- **Always communicate in Spanish** with the user
- All explanations, discussions, and documentation in Spanish

## Code and Comments

- **All code must be in English**
- **All comments must be in English**
- **All commit messages must be in English**

## Production Standards

- **100% Production Environment**
- **NO fake data, demo data, mockups, or false information**
- All content must be completely real and validated
- Professional quality content only
- Rigorous and accurate information required

## Development Workflow

- Follow the OpenSpec protocol for specifications (https://github.com/Fission-AI/OpenSpec)
- Use proper version control with Git
- Maintain comprehensive documentation
- Implement proper testing before deployment

## Technology Stack

### Frontend
- React with Vite
- Material UI M3 (Material Design 3)
- No Tailwind CSS
- No TypeScript

### Backend
- Laravel (PHP)
- MySQL database
- REST API architecture

## Content Standards

- Kin descriptions: equivalent to a full A4 page with detailed explanations
- Professional and didactic content
- Multiple illustrations where appropriate
- Extensive and relevant information

## User Types

- Anonymous users: access basic content
- Registered users: additional features (save dates, reminders, etc.)
- Both casual and continuous users
- Scholars and non-scholars

## Authentication

- Email registration with OTP verification
- Google OAuth integration
- Secure authentication (JWT or sessions)

## Design Principles

- Fully responsive (mobile, tablet, desktop)
- Scalable architecture for future i18n
- Material Design 3 guidelines
- Professional and accessible interface

## Hosting

- VPS self-hosted
- Custom domain
- Future-proof for expansion phases
